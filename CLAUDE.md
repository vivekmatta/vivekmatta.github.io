# Claude Instructions

## Git Workflow
- Regularly save and commit changes to git throughout every session.
- Write organized, descriptive commit messages that clearly explain what changed and why.
- Push commits to the remote repository frequently so all work is backed up and reversible.
- Group related changes into a single commit rather than committing unrelated changes together.
- Before starting any significant change, ensure the working tree is clean and up to date (`git pull`).
- This ensures we can always revert to a previous version if needed.
